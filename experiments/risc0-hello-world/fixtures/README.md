# RISC Zero Groth16/BN254 Fixtures

Generated by `src/bin/dump_groth16.rs` — proves `17 × 23 = 391` using
`ProverOpts::groth16()` (RISC Zero zkVM 3.0.5, risc0-circuit-recursion 4.0.4).

## Files needed for BN254 Groth16 verification

These three form the `(proof, vk, public_inputs)` triple to be consumed by
`groth16.Verify()` in the BN-to-BLS wrapper circuit (presumably gnark).

| File | Size | Content |
|------|------|---------|
| `seal.bin` | 256 bytes | Groth16 proof: A∈G1 (64 B) \| B∈G2 (128 B) \| C∈G1 (64 B), all coordinates big-endian 32-byte integers |
| `vk.json` | — | Groth16 verifying key in snarkjs-compatible JSON (`protocol: groth16, curve: bn128, nPublic: 5`). Hardcoded from `risc0-groth16 3.0.4` verifier constants — system-wide, not per-program |
| `public_inputs.json` | — | The 5 BN254 Fr elements passed to the verifier alongside the seal (see derivation below) |

## Files documenting how public inputs are derived

| File | Size | Content |
|------|------|---------|
| `claim_digest.bin` | 32 bytes | SHA-256 tagged hash of `ReceiptClaim` — proof-specific; encodes the entire execution result (pre/post state, exit code, journal). Source of `public_inputs[2]` and `[3]` |
| `journal.bin` | 8 bytes | Raw public output written by the guest (`391u64` little-endian). Embedded inside `ReceiptClaim.output.journal`, which feeds into `claim_digest` |
| `image_id.bin` | 32 bytes | Hash of the guest ELF (8 × u32 LE words). Identifies which program ran; encoded inside `ReceiptClaim` and therefore inside `claim_digest` |
| `control_root.bin` | 32 bytes | `ALLOWED_CONTROL_ROOT` — Merkle root of the allowed RISC Zero recursion programs. Fixed for risc0-circuit-recursion 4.0.4. Source of `public_inputs[0]` and `[1]` |
| `bn254_control_id.bin` | 32 bytes | `BN254_IDENTITY_CONTROL_ID` — fingerprint of the final STARK→Groth16 compression program. Fixed for risc0-circuit-recursion 4.0.4. Source of `public_inputs[4]` |

## Public input derivation

```
(inputs[0], inputs[1]) = split_digest(control_root)
(inputs[2], inputs[3]) = split_digest(claim_digest)
 inputs[4]             = Fr(reverse_bytes(bn254_control_id))
```

`split_digest(d)`: reverse `d` to big-endian, split at byte 16 →
return `(Fr(bytes[16..32]), Fr(bytes[0..16]))` as zero-padded
32-byte big-endian hex strings.

`inputs[0,1,4]` are **version constants** — identical for all valid
RISC Zero 3.x proofs. `inputs[2,3]` are **proof-specific** — they
change per proof because `claim_digest` encodes the actual execution
(journal output, pre/post state, image ID).
